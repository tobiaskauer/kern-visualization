# KERN Visualization

KERN Visualization ist eine Sammlung von Datenvisualisierungskomponenten für das KERN UX Design System. Die Komponenten lesen KERN CSS Custom Properties direkt aus — sie spiegeln automatisch das aktive Theme (Hell/Dunkel) wider, ohne zusätzliche Konfiguration.

Gebaut mit **TypeScript** und **D3.js**, kein Framework erforderlich.

## Schnellstart

```ts
import { BarChart } from 'kern-visualization';

const chart = new BarChart({
  container: document.getElementById('chart')!,
  data: [
    { label: 'Jan', value: 42 },
    { label: 'Feb', value: 67 },
    { label: 'Mär', value: 55 },
  ],
  title: 'Monatliche Umsätze',
  xAxisLabel: 'Monat',
  yAxisLabel: 'Umsatz (€)',
  animated: true,
  gridlines: { y: true },
});

chart.render();
```

Alle Diagramme folgen demselben Muster: Instanz mit Konfiguration erstellen, `.render()` aufrufen. Beim Anpassen von Parametern `.update()` aufrufen oder eine neue Instanz erstellen.

## Farbschemata

Drei Paletten stehen zur Verfügung, alle aus den KERN Design Tokens abgeleitet:

- **`categorical`** — 8 distinkte Farben für ungeordnete Kategorien (Standard)
- **`sequential`** — Blaureihe von hell nach dunkel für ordinale oder gerankte Daten
- **`diverging`** — Türkis → Neutral → Rot für Werte um einen Mittelpunkt (z. B. Abweichungen vom Zielwert)

Das Farbschema reagiert automatisch auf das aktive KERN-Theme (Hell/Dunkel).

---

## Responsivität

Diagramme passen sich automatisch der Containerbreite an. Die Mindestbreite beträgt 280 px — wird dieser Wert unterschritten, wird das Diagramm in einem horizontal scrollbaren Bereich dargestellt. Ein `ResizeObserver` sorgt dafür, dass das Diagramm bei Größenänderung des Containers automatisch neu gerendert wird.

## Barrierefreiheit

KERN Visualization folgt den sieben Prinzipien des [Chartability-Frameworks](https://chartability.fizz.studio/) — einem Prüfrahmen speziell für Datenvisualisierungen, der über den allgemeinen WCAG-Standard hinausgeht.

### POUR + CAF

Die sieben Prinzipien gliedern sich in zwei Gruppen:

**POUR** — die vier WCAG-Grundprinzipien, auf Datenerfahrungen übertragen:

- **Wahrnehmbar** — Informationen müssen ohne Rücksicht auf ein bestimmtes Sinnesorgan zugänglich sein (Sehen, Hören, Berühren).
- **Bedienbar** — Navigations- und Interaktionsmechanismen müssen für alle Eingabemodalitäten funktionieren (Maus, Tastatur, Touch).
- **Verständlich** — Inhalt und Bedienung müssen ohne Vorannahmen über das Vorwissen der Nutzenden verständlich sein.
- **Robust** — Die Darstellung muss mit aktuellen und zukünftigen Hilfstechnologien (Screenreader, Browser-Erweiterungen) zuverlässig funktionieren.

**CAF** — drei Erweiterungsprinzipien, die spezifisch für komplexe Datenerfahrungen gelten:

- **Kompromittierend** — Informationen müssen über mehrere gleichwertige Wege erreichbar sein (z. B. Diagramm und Datentabelle) und die Transparenz über Datenquellen gewährleisten.
- **Unterstützend** — Die Oberfläche soll kognitive Last aktiv reduzieren: Muster und Trends explizit benennen, Datendichte kontrollieren, Navigation vereinfachen.
- **Flexibel** — Nutzende sollen persönliche Einstellungen (Kontrast, Schriftgröße, Animationen, Farbpräferenzen) durchsetzen können, ohne dass das Diagramm diese überschreibt.

Die folgende Tabelle zeigt, was die Bibliothek automatisch bereitstellt und was Entwickler:innen selbst umsetzen müssen.

## Small Multiples

Für den Vergleich mehrerer Facetten steht die `SmallMultiples`-Komponente zur Verfügung:

```ts
import { SmallMultiples } from 'kern-visualization';

new SmallMultiples({
  container: document.getElementById('grid')!,
  type: 'BarChart',
  data: flatData,
  facetKey: 'region',
  columns: 3,
}).render();
```

Skalen werden automatisch über alle Facetten hinweg synchronisiert.
